# Project Instructions

## Overview

Monorepo with two Astro sites deployed to Cloudflare Workers:

- **landing/** — Marketing landing site with i18n (EN/ES), blog, and SEO
- **docs/** — Documentation site (Astro Starlight, bilingual EN/ES)

## Tech Stack

- **Framework**: Astro (v5.x for docs, v6.x for landing)
- **Package manager**: Bun (each project has its own `bun.lock`)
- **Language**: TypeScript (strict config via `astro/tsconfigs/strict`)
- **Styling**: Tailwind CSS v4 (landing), CSS (docs)
- **Content**: Astro Content Collections with Markdown/MDX
- **Deployment**: Cloudflare Workers via Wrangler (static assets in `./dist`)

## Commands

All projects share the same commands, run from their directory:

```bash
bun dev       # Start dev server
bun run build # Build to ./dist (NOT `bun build` — that invokes Bun's bundler)
bun preview   # Preview production build
```

## Conventions

### Naming

- **Files/directories**: kebab-case
- **Astro components**: PascalCase (e.g., `BaseLayout.astro`)
- **TypeScript/JS**: camelCase

### Indentation

- `.astro` page files: tabs
- `.ts`/`.js` files: 2 spaces

### Content

- **Docs**: MDX files in `docs/src/content/docs/`. English at root, Spanish under `es/`.
- **Releases**: MDX files in `docs/src/content/docs/versions/`. File names use underscores for version numbers (e.g., `2_0.mdx`).
- **Blog**: Markdown files in `landing/src/data/blog/{locale}/`. Each post requires frontmatter with `title`, `description`, `pubDate`, `author`, `locale`, and optional `image`.

### i18n (Landing)

- All URLs are prefixed with locale: `/en/`, `/es/`
- Locales defined in `landing/src/utils/i18n.ts` and `landing/astro.config.mjs`
- Page content loaded via dynamic imports from `landing/src/data/{locale}/`
- Blog posts filtered by `locale` field in frontmatter
- Navigation language switcher preserves the current path

### Adding a new language (Landing)

1. Add locale to `locales` array in `src/utils/i18n.ts`
2. Add to `localeNames` in the same file
3. Add to `astro.config.mjs` → `i18n.locales` and sitemap config
4. Create `src/data/{locale}/` with `home.ts`, `layout.ts`, `blog.ts`
5. Add blog posts in `src/data/blog/{locale}/`

### SEO (Landing)

- `BaseLayout` accepts `title`, `description`, `image` props for meta tags
- Every page gets: canonical URL, hreflang alternates, Open Graph, Twitter Cards
- Blog posts include JSON-LD Article structured data
- Sitemap generated by `@astrojs/sitemap` with i18n locale mapping
- `robots.txt` in `public/`

### Assets

External assets are hosted at `https://xer-assets.juanrcifuentes.com/`. Reference them by URL, don't store them locally.

### Structure

Each site follows standard Astro layout:

```
landing/
├── src/
│   ├── components/       # Reusable .astro components
│   ├── data/
│   │   ├── en/           # English content (home.ts, layout.ts, blog.ts)
│   │   ├── es/           # Spanish content (same structure)
│   │   └── blog/         # Blog posts (content collection)
│   │       ├── en/       # English posts (.md)
│   │       └── es/       # Spanish posts (.md)
│   ├── layouts/          # BaseLayout with SEO meta tags
│   ├── pages/
│   │   ├── index.astro   # Root redirect → /en/
│   │   └── [locale]/     # Dynamic locale pages
│   │       ├── index.astro
│   │       └── blog/
│   ├── utils/            # i18n helpers
│   ├── content.config.ts # Blog collection schema
│   └── styles.css        # Tailwind + Typography plugin
├── public/               # robots.txt, static files
├── astro.config.mjs      # i18n routing, sitemap, site URL
├── wrangler.toml
└── package.json
```

## Important Notes

- No test framework is configured. Verify changes with `bun run build`.
- Both sites are statically generated.
- There is no root `package.json` or workspace config — each project is fully independent with its own `bun.lock` and `node_modules`. Run `bun install` inside each project directory.
- Landing site URL: `https://xer.juanrcifuentes.com` (configured in `astro.config.mjs` and `robots.txt`)
- Landing uses `@tailwindplus/elements` for interactive components (dialog, popover)
